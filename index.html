What Came First?

Years ago, we decided on Quick C instead of Quick Basic. Basic offered a quicker reward in that MS-DOS came with a BASIC interpreter so familiarity was there. All that poking and peeking, sort of put us off so we went with C. Steeper learning curve but capable of inline assembler, which is what you needed for graphics drivers.
We didn't have a goal, just interest. A decade later when most Windows development was Delphi or Basic our C background kicked in. We got a contract writing a virtual disk driver because of the way we wrote an in house program featuring multiple documents displaying OpenGL using Microsoft Foundation Classes (MFC). Never did make any money with graphics.
